## CRITICAL (block GTM)

- **[app/api/cpa/assign/route.ts:9]** Missing role check. Any authenticated user (secretary, clinic_owner) can POST to this endpoint and assign any CPA to any clinic. Only a platform admin or the clinic_owner should be able to do this. The only check is `if (!user)` — there is no `user.role` gate. An attacker with a valid session can exfiltrate another clinic's financial data by assigning a CPA account they control.

- **[app/api/reminders/process/route.ts:81]** Cross-tenant data processing. When triggered by an authenticated secretary (non-cron path), the `findMany` query has no `clinicId` filter — it processes every pending reminder across all clinics. A secretary at Clinic A can trigger Clinic B's patient reminder sends. The query should be scoped to `user.clinicId` on the non-internal path.

- **[app/api/send-receipt/route.ts:10]** Internal error message returned to client. `err.message` from a Prisma/Resend exception is serialised directly into the 500 response. Production exception messages expose DB schema names, connection strings, or internal paths depending on the error source. Use a generic message for the client and log the real error server-side.

- **[.env.local (not committed, but present on disk)]** Real production credentials on disk: `SUPABASE_SERVICE_ROLE_KEY` (full DB bypass), `DATABASE_URL` with plaintext password `.TwentyTwo22.`, and anon key. The `.gitignore` pattern correctly excludes `.env.*`, and git history confirms no accidental commit. However, the password `.TwentyTwo22.` is a dictionary-adjacent value. Rotate it and use a strong random password. Document that `.env.local` must never be transferred in plaintext (email, Slack, etc.). CWE-259.

## HIGH (fix before launch)

- **[40/46 API routes]** No `try/catch` around any route handler. If Prisma throws (connection drop, constraint violation, malformed UUID), Next.js returns a 500 with a default error response that may include stack traces in development mode or when `NODE_ENV` is not strictly `production`. Wrap all route handlers. Affected routes include the entire payroll, expense, appointment, supplier, and report surfaces.

- **[app/(auth)/login/actions.ts]** No application-level rate limiting on the login endpoint. Supabase provides some built-in rate limiting, but it is project-configurable and defaults may be loose. There is no IP-level throttle, no lockout after N failures, and no CAPTCHA. CWE-307.

- **[app/api/reminders/cron/route.ts:7]** Guard has a logic flaw: `if (!process.env.CRON_SECRET || auth !== ...)`. If `CRON_SECRET` is unset (it is absent from `.env.local`), the condition short-circuits to `true` and returns 401 — which means reminders never fire in that environment. But if a deployment accidentally sets `CRON_SECRET=""`, the empty-string comparison would reject all requests forever. The real issue is this env var is missing from `.env.local` with no fallback, meaning cron is silently broken in the current local environment.

- **[app/api/reports/quarterly/route.ts:301]** `reason: error.message` from an internal Resend API call is returned to the client. Same class of issue as send-receipt. CWE-209.

- **[app/(dashboard)/layout.tsx]** The dashboard layout redirects unauthenticated users, but it does not check `user.role`. A CPA user who navigates directly to `/patients` or `/payments` will pass the `!authUser` guard and reach the page (the server actions behind them do re-check via `getActorDb`, but the page-level data loads do not consistently enforce role). Audit each dashboard page's `page.tsx` to confirm every data-fetching call goes through `getActorDb`/`withClinicDb` and not the bare `prisma` client.

## MEDIUM (fix within 30 days)

- **[app/api/cpa/reports/[clinicId]/csv/route.ts:20]** The `period` query parameter is passed through `parsePeriod()` which handles invalid input safely, but the parameter is also used unsanitised in the `Content-Disposition` filename: `filename="invoices-${period || 'report'}.csv"`. A crafted `period` value containing `"` or `;` could inject a second Content-Disposition directive in some browsers. Sanitise to alphanumeric + `-` before use in the header.

- **[middleware.ts:60-65]** The Vercel preview bypass (`host.endsWith('.vercel.app')`) skips all routing and auth logic for any `.vercel.app` domain. If a preview deployment is shared with an external party, that party can access the full app without subdomain routing enforcement. This is intentional for dev convenience but should be documented as a known gap, and preview deployments should require Vercel authentication.

- **[app/api/cpa/assign/route.ts — no input validation]** `cpaUserId` and `clinicId` are passed directly to Prisma without UUID format validation. A non-UUID string will cause a Prisma runtime error that (per the missing try/catch above) propagates as an unhandled 500.

- **[lib/admin.ts — ADMIN_EMAILS env var]** The `/admin` route is gated by email membership in a comma-separated env var. If `ADMIN_EMAILS` is unset, `isAdminEmail()` returns false for everyone — the admin panel is inaccessible but not exploitable. The real risk is if the env var is accidentally set to `""` or `"*"` during a deployment. Validate the format on startup.

## Refactoring Candidates

- **[app/(dashboard)/payments/actions.ts — confirmPayment, lines 282–500+]** ~220 lines doing VAT math, OR number generation, loyalty card creation, card use decrementation, invoice creation, SC/PWD audit logging, and recall reminder scheduling — at least six distinct responsibilities. Extract each into a named helper. This is the highest-value financial path and currently impossible to unit test.

- **[app/(dashboard)/patients/actions.ts — 600 lines]** The file mixes patient CRUD, consent tracking, anonymisation, and loyalty card lookups. Split into at least two modules.

- **[parsePeriod() duplicated]** Identical `parsePeriod()` function defined in both `app/api/cpa/reports/[clinicId]/_lib.ts` and `app/(cpa)/clinics/[clinicId]/page.tsx`. Extract to a shared util.

- **[app/(dashboard)/patients/actions.ts and app/api/patients/search/route.ts]** Patient search logic exists in two places with slightly different shapes (one uses `withClinicDb`, one uses both `prisma` + `withClinicDb`). Consolidate into one canonical search function.

- **[Dead SMS branch — app/api/reminders/process/route.ts:100-115]** The SMS channel case is implemented as a permanent `FAILED` mark with a Phase 2 TODO. If secretaries are selecting SMS as a reminder channel, their reminders silently fail forever. Either remove the channel from the UI or surface an explicit "not yet available" warning.

## Missing Tests

- **No test suite exists.** `find` returns zero `.test.ts` / `.spec.ts` files. There is no Jest, Vitest, or Playwright config. This is a complete gap.

- **confirmPayment [payments/actions.ts:282]** — The core billing function. VAT calculation, OR number sequencing, loyalty card use decrementation, and SC/PWD discount are all untested. A rounding or sequencing bug here is a BIR compliance violation.

- **withClinicDb / RLS enforcement [lib/clinic-db.ts]** — The entire multi-tenancy security model depends on `set_config('app.clinic_id', …, TRUE)` being called before every query. This is untested. A future refactor that accidentally bypasses it would be invisible without a cross-tenant isolation test.

- **verifyCpaAccess [app/api/cpa/reports/[clinicId]/_lib.ts]** — The gate that prevents one CPA from reading another clinic's financials. No test that verifies an unassigned CPA gets 403.

- **parsePeriod [_lib.ts]** — Parses BIR quarter strings. Invalid input silently defaults to current month. Edge cases (year boundary, Q4 wrap) are untested.

- **Auth flows (login, reset-password, onboarding)** — No tests for the happy path, let alone invalid token or expired session edge cases.

## Dependency Risks

- **next@14.2.35** — Next.js 14 is not the current major (15 is current). 14.2.x has received security patches; check the Next.js changelog for any CVEs since 14.2.0. The `^` range in `eslint-config-next` would follow minor bumps but not major. Pin to a specific patch if staying on 14.

- **lucide-react@^1.16.0** — Version `1.x` is a pre-release major bump released in 2025/2026. It was likely AI-suggested since the stable ecosystem uses `^0.x`. Verify this is the intended version; `0.x` is what most shadcn/ui tooling expects and there may be icon name changes.

- **All `^` ranges** — Every dependency uses `^` (loose minor/patch range). For a production SaaS handling patient SPI and financial records, pin exact versions in `package.json` and use `package-lock.json` for reproducibility. A compromised minor release of any dependency (e.g. `resend`, `@supabase/ssr`) would auto-install on the next `npm install`.

- **thermal-printer-encoder@^3.0.0** — Niche package, low download count, unclear maintenance status. Verify it is actively maintained and has no known CVEs before shipping to production clinics.

- **npm audit unavailable** — The registry mirror (`npmmirror.com`) does not implement the audit endpoint. Switch the npm registry back to `https://registry.npmjs.org` and run `npm audit` before launch to surface any known CVEs.

## What Looks Good

- **HMAC-SHA256 webhook verification** on the Messenger endpoint (`app/api/messenger/webhook/route.ts`) uses `timingSafeEqual` with a length guard — this is correct and uncommon. Well done.
- **`withClinicDb` / RLS architecture** is the right pattern for multi-tenant isolation. The design — transaction-local `set_config` before every query, with a NOBYPASSRLS role — is sound and documented.
- **Admin panel** requires email allowlist + Supabase session check in both `page.tsx` and `actions.ts`. Double-checked.
- **No `dangerouslySetInnerHTML`** anywhere in the codebase — XSS via React is not a concern.
- **No hardcoded secrets in git history** — the only `.env` ever committed held the Prisma scaffold placeholder, which was subsequently removed and gitignored.
- **Password handling is fully delegated to Supabase Auth** — no application-level password hashing, so there is no opportunity for CWE-916 weak hashing.
- **CSV output properly escapes quotes** (`replace(/"/g, '""')`) — no CSV injection.
